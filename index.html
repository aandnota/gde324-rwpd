Project 5

Responsive Portfolio Site

The Present and The Future

SUMMARY

Responsive Design is the present and the future. We will be building a responsive site (mobile-first) that showcases the work you’ve done in this class.

READING

Read the following:

Responsive Web Design, Ethan Marcotte, 2010, A List Apart

What We Mean When We Say Responsive, Lyza Danger Gardner, 2014, A List Apart

Do As Little As Possible, Lyza Danger Gardner, 2013, A List Apart

ASSIGNMENT

Design and build a site that is mobile-first, but that also scales and adapts itself for different browser-widths.

You should use the following techniques:

  • Fluid HTML
  • Fluid Images
  • Media Queries in your CSS

Your site should also include the following:

  • At least 5 Images that scale appropriately
  • At least 5 distinct sections on the home page
  • At least 5 pages that are extensions of the sections (including the home page)
  • A footer

Your site should look good at the following resolutions in both portrait and horizontal orientations:

  • 320x640
  • 768x1024
  • 1440x900

TOOLS

For this project we will be using the following tools:

  • Photoshop/Illustrator - for designing layouts or preparing images

Sublime Text 3

For Sublime there are a couple of plugins (called "packages") that you'll want to install:

  • Emmet
  • HTML5
  • CSS3

We’ll get Sublime setup in class, so if you’re not there or you have any questions, please find me. This part can be tricky, but makes coding HTML/CSS so much easier.

GitHub Client for Mac or Windows

This is what we'll primarily use for pushing (sending) HTML/CSS files to the web.

PHASES

  1. Research

    1. Create and collect images of your work from this semester
      1. You should have at least 5 images.
    2. Each image should be at least 1200px wide
      1. It'll be easier to arrange on the page if all of your images are of a similar proportion (aspect ratio)

  2. Prototype

    1. Design and develop your pages in Sublime Text 3
    2. Remember, this website is mobile first.
    3. Think of the larger resolutions as the adaptation sof your mobile design.

  3. Final Design

    1. After initial feedback, re-design or fix issues.
    2. Submit final version.
