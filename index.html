Project 6

Responsive Portfolio Site

The Present and The Future

SUMMARY

Responsive Design is the present and the future. We will be building a responsive site (mobile-first) that showcases the work you’ve done in this class.

READING

Read the following:

HTML/CSS Basics, MIT, ignore the bits at the end about their class meetings

Responsive Web Design, Ethan Marcotte, 2010, A List Apart

What We Mean When We Say Responsive, Lyza Danger Gardner, 2014, A List Apart

Do As Little As Possible, Lyza Danger Gardner, 2013, A List Apart

ASSIGNMENT

Part 1

Install Atom.io

Part 1a

Next, for practice and before you start on your portfolio site, scrape a Wikipedia page and redesign it. Include all of the text, images, and sections evident on the source page.

Part 2A

Design and build a site that is mobile-first, but that also scales and adapts itself for different browser-widths.

The first part is the purely design phase and should be done in Adobe XD.

Part 2B

This phase is done after you complete the first part.

You should use the following techniques:

  • Flexbox
  • Fluid Images
  • Media Queries in your CSS

Your site should also include the following:

  • At least 5 images that scale fluidly.
  • At least 5 distinct sections on the home page.

Your site should look good at the following resolutions in both portrait and horizontal orientations:

  • 320x640
  • 768x1024
  • 1440x900

TOOLS

For this project we will be using the following tools:

  • A piece of paper - for designing layouts
  • Photoshop/Illustrator or some other kind of digital drawing application.

Atom.io

For Atom there are a couple of plugins (called "packages") that you'll want to install:

  • Emmet
  • HTML5
  • CSS3

We’ll get Atom setup in class, so if you’re not there or you have any questions, please find me. This part can be tricky, but makes coding HTML/CSS so much easier.

Chrome

This is the primary browser we'll be using to test and debug our sites. You'll also want to view and test your sites in Firefox and Safari (at least), but Chrome is the one that makes debugging the easiest.

Firefox

Flexbox Froggy is a fun, little website to help you wrap your noggins around Flexbox.

PHASES

  1. Research

    1. Create and collect images of your work from this semester
      1. You should have at least 5 images.
    2. Each image should be at least 1200px wide
      1. It'll be easier to arrange on the page if all of your images are of a similar proportion (aspect ratio)

  2. Prototype

    1. Design in Sketch/XD and develop your pages in Atom.io
    2. You need one page, with five clearly delineated sections.
    3. Remember, this website is mobile first.
    4. Think of the larger resolutions as the adaptations of your mobile design.
    5. (OPTIONAL) Once you've coded a webpage, we will shift to Adobe XD to build an interactive prototype of a website, based off the design of the first webpage.

  3. Final Design

    1. After initial feedback, re-design or fix issues.
    2. Submit final version of your code and Adobe XD file.
